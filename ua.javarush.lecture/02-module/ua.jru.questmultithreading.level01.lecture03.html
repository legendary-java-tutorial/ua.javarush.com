— Сьогодні ми вивчатимемо клас Object.
Ти вже стикався з ним і знаєш, що Object – базовий клас для всіх класів. У нього немає жодних даних, але є кілька методів.

- А навіщо йому методи? Хіба хтось творить його об'єкти?

— Подивися на ситуацію з такого боку – методи, які є у Object'у, є взагалі у всіх класів. Тобто. розробники Java відібрали кілька методів, які, на їхню думку, мають бути у всіх класів та додали їх до класу Object.

А у поєднанні з поліморфізмом – можливістю перевизначати методи класу Object у класах-спадкоємцях – це породжує дуже потужний інструмент.

Давай подивимося, що це за методи:

- Ок. Я уважно слухатиму.

- Почнемо з методу toString();

Цей метод дозволяє отримати текстовий опис будь-якого об'єкта. Реалізація його в класі Object дуже проста:

return getClass().getName() + "@" + Integer.toHexString(hashCode());

getClass() і hashCode() – це також методи класу Object.
Ось стандартний результат виклику такого методу

java.lang.Object@12F456

— І в чому користь такого опису?

— З такого опису можна дізнатися про клас об'єкта, у якого викликали даний метод. А також можна розрізняти об'єкти – різним об'єктам відповідають різні цифри після символу @.

Але цінність цього методу в іншому. Даний метод можна перевизначити в будь-якому класі і повертати більш потрібний або детальніший опис об'єкта.

Але це ще не все. Завдяки тому, що для кожного об'єкта можна отримати його текстове подання, Java можна було реалізувати підтримку «складання» рядків з об'єктами.
Ось дивись:

Код Що відбувається насправді 
int age = 18;
