Для чого використовується YAML

Ще один текстовий формат даних - YAML ( Yet Another Markup Language , пізніше - YAML Ain't Markup Language ). Використовується для серіалізації об'єктів для їх передачі через мережу (так само, як використовується xml та json). За рахунок зручної читання його також використовують для написання конфігураційних файлів, наприклад, для Docker, Kubernetes, Ansible, etc. Якщо YAML потрібно зберегти у файл (наприклад, конфігураційний), використовують два розширення: .yaml і .yml.

Синтаксис мови

Для позначення тегів xml використовують кутові дужки ( <> ), для JSON - фігурні ( {} ). Для YAML застосовують роздільники рядки (Enter) і відступи (пробіли чи табы).

Дані зберігаються у вигляді key-value пар, де ключем виступає рядок, а значення можуть бути різні типи даних: рядок, число, true/false, масив... Ключі пишуться без лапок.

Давай послідовно подивимося, як у YAML зберігається інформація:

Як і Java, елементом списку може бути список, тобто об'єкти можна вкладати один в інший. Дефіс, який визначає черговий елемент списку, може бути зміщений по горизонталі щодо батьківського ключа або знаходиться безпосередньо під ним. Головне – щоб усі елементи мали однаковий формат. Є ще два нюанси під час роботи з текстовими значеннями:

  1. Багаторядковий текст. Ми можемо зберегти текст так:

    multilineText: "line 1\nline 2\n....line n"

    Але читати його буде дуже незручно. Тому є символ | (pipe), за допомогою якого можна записати текст інакше:

    multilineText: |
     line 1
     line 2
     ....
     line n

    Погодься, другий варіант зручніше, вірно?

  2. Довгі рядки. Якщо текст потрібно зберегти в один рядок, але при цьому хочеться, щоб він містився у видимій робочій області IDE, можна використовувати символ > (greater).

    singlelineText: >
     begin
     ...
     continue same line
     ...
     end

    Весь текст буде сприйнятий як один рядок.

Якщо є необхідність кілька YAML-структур даних записати в один файл, тоді між ними потрібно використовувати роздільник ( три дефісу). Насправді потреба зустрічається рідко, але краще знати про такому варіанті.

Приклад YAML-документа

Давай створимо в Java якусь структуру даних (клас) та об'єкт цього типу – і спробуємо уявити дані цього об'єкта у серіалізованому вигляді у форматі YAML.

class Family {
   private Date weddingDate;
   private Person wife;
   private Person husband;
   private List<Person> children;

   // getters and setters are omitted
}

class Person {
   private final String name;
   private final boolean isWoman;
   private int age;

   public Person(String name, int age, boolean isWoman) {
       this.name = name;
       this.age = age;
       this.isWoman = isWoman;
   }

// getters and setters are omitted

}

public static void main(String[] args) {
   Person wife = new Person("Anya", 37, true);
   Person husband = new Person("Alex", 40, false);
   var children = List.of(
           new Person("Inna", 12, true),
           new Person("Olya", 5, true)
   );
   Date weddingDate = new Date(/*some long*/);

   Family family = new Family();
   family.setWeddingDate(weddingDate);
   family.setWife(wife);
   family.setHusband(husband);
   family.setChildren(children);
}

   age: 12
   isWoman: true
 - name: Olya
   age 5
   isWoman: true
