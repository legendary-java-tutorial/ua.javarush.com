- Привіт, Аміго! Я дивлюся, ти робиш великі успіхи у вивченні ниток.

— Це виявилося нескладно.

- Це ж чудово! Сьогодні легкий урок і тема цього уроку – метод  join .

Уяви собі ситуацію: головна нитка створила дочірню нитку для виконання якогось завдання. Минає час, і ось головній нитці знадобабося результати роботи тієї дочірньої нитки. А дочірня нитка ще не закінчила своєї роботи. Що робити головною ниткою?

— Так що робити головній нитці?

- Для цього є метод  join . Сенс його в наступному. Одна нитка чекає, доки повністю завершиться робота другої нитки:

Код Опис 
class Printer implements Runnable
{
private String name;
public Printer(String name)
{
this.name = name;
}
public void run()
{
System.out.println("I’m " + this.name);
}
}
 Клас, який реалізує Runnable інтерфейс. 
public static void main(String[] args)
{
Printer printer1 = new Printer("Коля");
Thread thread1 = new Thread(printer1);
thread1.start();

thread1.join();
}
 Головна нитка створює дочірню нитку – об'єкт  thread1 .

Потім запускає її виклик  thread1 .start();

І чекає на її завершення –  thread1.join();

Одна нитка може викликати метод  join  у об'єкта другої нитки.  У результаті перша нитка (яка викликала метод) припиняє свою роботу до закінчення роботи другої нитки (об'єкт якої був викликаний метод).

Тут варто розрізняти дві речі: є, власне, нитка – окремий процес виконання команд, а є об'єкт цієї нитки (об'єкт Thread).

- І це все?

- Так.

— А навіщо треба створювати нитку і одразу ж чекати на її завершення?

— Одразу ж може й не треба. А ось через якийсь час це може й знадобиться. Головна нитка після запуску першої дочірньої нитки може  роздати ще багато завдань  іншим ниткам (створивши їх і викликавши метод  start ), а потім все – роботи їй більше не залишилося, потрібно опрацьовувати результати роботи першої дочірньої нитки. У таких випадках, коли потрібно дочекатися завершення роботи іншої нитки і потрібно викликати метод  join .
