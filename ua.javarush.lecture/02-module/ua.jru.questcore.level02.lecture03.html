- Привіт, Аміго! А я розповім тобі про  навантаження методів.

- Їх ще й перевантажувати можна? Та що за день сьогодні такий!

— З ними багато чого можна робити, але не про це будемо.

- Згоден.

- Перевантаження - це дуже проста операція.  Насправді це навіть не операція над методами, хоча іноді її називають страшним словом -  параметричний поліморфізм.

Справа в тому, що всі методи всередині класу повинні мати унікальні імена.

- Так, я знаю про це.

— Так ось це не зовсім так. Точніше зовсім не так. Метод повинен мати унікальне ім'я. Унікальним має бути  об'єднання з імені та типів параметрів  цього методу. Їх ще називають сигнатурами методів.

Код Опис 
public void print();
public void print2();
 Так можна. Два методи мають унікальні імена. 
public void print();
public void print(int n);
 І так можна. Два методи мають унікальні імена (сигнатури). 
public void print(int n, int n2);
public void print(int n);
 Досі унікальні методи. 
public int print(int a);
public void print(int n);
 А так не можна. Методи не унікальні, хоч і повертають різні типи. 
public int print(int a, long b);
public long print(long b, int a);
 А так – можна. Параметри методів є унікальними.

— Десь я вже таке бачив.

- Ага. Коли ти пишеш System.out.println, Intellij IDEA підказує тобі і видає у підказці кілька десятків методів print із різними параметрами. Компілятор просто визначить потрібний метод за типами змінних, які ти туди передаєш, і пропише виклик саме його.

— Це начебто не складно. Не поліморфізм, однак.

- Точніше - не перевизначення методів

До речі, звертаю твою увагу, що  імена параметрів ролі не відіграють  – вони губляться під час компіляції. Після компіляції про метод відомо  тільки його ім'я та типи параметрів.
