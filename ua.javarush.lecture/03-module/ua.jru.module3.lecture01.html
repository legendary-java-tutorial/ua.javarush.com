Дана програма – імітація іподрому

Для початку, як і в минулому проекті, зроби собі форк із репозиторію: https://github.com/pavlo-plynko/hippodrome , і цей форк схиляй собі.

Твоє завдання – додати тестування та логування.

Список необхідних тестів

У наступному списку кожен пункт потрібно реалізувати як тестовий метод. Вигадуючи імена тестовим методам, намагайся бути лаконічним, але в той же час, щоб можна було приблизно зрозуміти, що саме в них тестується. Наприклад, порівняй ці два результати тестування:

У другому випадку важче зрозуміти, що за тести не пройшли і доведеться дивитися логи.

1. Клас Horse:

  1. конструктор
    • Перевірити, що при передачі в конструктор першим параметром nullбуде викинуто IllegalArgumentException. Для цього потрібно скористатися методом assertThrows;
    • Перевірити, що при передачі в конструктор першим параметром null, викинутий виняток міститиме повідомлення "Name cannot be null.". Для цього потрібно отримати повідомлення з перехопленого виключення та скористатися методом assertEquals;
    • Перевірити, що при передачі в конструктор першим параметром порожнього рядка або рядка містить лише пробілові символи (пробіл, табуляція і т.д.), буде викинуто IllegalArgumentException. Щоб виконати перевірку з різними варіантами пробілів, потрібно зробити тест параметризованим;
    • Перевірити, що при передачі в конструктор першим параметром порожнього рядка або рядка, що містить лише пробілові символи (пробіл, табуляція і т.д.), викинутий виняток міститиме повідомлення "Name cannot be blank.";
    • Перевірити, що з передачі в конструктор другим параметром негативного числа, буде викинуто IllegalArgumentException;
    • Перевірити, що при передачі в конструктор другим параметром від'ємного числа, викинутий виняток міститиме повідомлення "Speed ​​cannot be negative.";
    • Перевірити, що з передачі в конструктор третім параметром негативного числа, буде викинуто IllegalArgumentException;
    • Перевірити, що при передачі до конструктора третім параметром від'ємного числа, викинутий виняток міститиме повідомлення "Distance cannot be negative.";
  2. метод getName
    • Перевірити, що метод повертає рядок, який був переданий першим параметром у конструктор;
  3. метод getSpeed
    • Перевірити, що метод повертає число, яке було передано другим параметром конструктор;
  4. метод getDistance
    • Перевірити, що метод повертає число, яке було передано третім параметром конструктор;
    • Перевірити, чи метод повертає нуль, якщо об'єкт був створений за допомогою конструктора з двома параметрами;
  5. метод move
    • Перевірити, що метод викликає всередині метод getRandomDoubleпараметрів 0.2 і 0.9. Для цього потрібно використовувати MockedStaticі його метод verify;
    • Перевірити, що спосіб надає дистанції значення вираховане за такою формулою: distance + speed * getRandomDouble(0.2, 0.9). Для цього потрібно замокати getRandomDouble, щоб він повертав певні значення, які потрібно задати параметризувавши тест.

2. Клас Hippodrome:

  1. Конструктор
    • Перевірити, що з передачі в конструктор null, буде викинуто IllegalArgumentException;
    • Перевірити, що при передачі в конструктор null, викинутий виняток міститиме повідомлення "Horses cannot be null.";
    • Перевірити, що з передачі в конструктор порожнього списку, буде викинуто IllegalArgumentException;
    • Перевірити, що при передачі до конструктора порожнього списку, викинутий виняток міститиме повідомлення "Horses cannot be empty.";
  2. метод getHorses
    • Перевірити, що метод повертає список, який містить ті ж об'єкти і в тій же послідовності, що список, який був переданий в конструктор. При створенні об'єкта Hippodrome передай у конструктор список із 30 різних коней;
  3. метод move
    • Перевірити, що метод викликає метод переміщення у всіх коней. При створенні об'єкта Hippodrome передай у конструктор список із 50 моків коней та скористайся методом verify.
  4. метод getWinner
    • Перевірити, що метод повертає коня з найбільшим значенням distance.

3. Клас Main

  1. метод main
    • Перевірити, чи метод виконується не довше 22 секунд. Для цього скористайся анотацією Timeout. Після написання цього тесту, відключи його (скористайся анотацією Disabled). Так він не займатиме час при запуску всіх тестів, а за необхідності його можна буде запустити вручну.

Що потрібно логувати

1. Клас Main:

  1. Після створення об'єкта іподрому, додати в лог запис виду:2022-05-31 17:05:26,152 INFO Main: Початок стрибків. Кількість учасників: 7
  2. Після виведення інформації про переможці, додати в лог запис виду:2022-05-31 17:05:46,963 INFO Main: Закінчення стрибків. Переможець: Вишня

Клас Hippodrome:

  1. Якщо конструктор був переданий null, то перед прокиданням виключення, додати в лог запис виду:2022-05-31 17:29:30,029 ERROR Hippodrome: Horses list is null
  2. b. Якщо конструктор був переданий порожній список, то перед прокиданням виключення, додати в лог запис виду:2022-05-31 17:30:41,074 ERROR Hippodrome: Horses list is empty
  3. Наприкінці конструктора додати у лог запис виду:2022-05-31 17:05:26,152 DEBUG Hippodrome: створення Hippodrome, коней [7]

3. Клас Horse:

  1. Якщо конструктор замість імені переданий null, то перед прокиданням виключення, додати в лог запис виду:2022-05-31 17:34:59,483 ERROR Horse: Name is null
  2. Якщо передане в конструктор ім'я порожнє, перед прокиданням виключення, додати в лог запис виду:2022-05-31 17:36:44,196 ERROR Horse: Name is blank
  3. Якщо передана в конструктор швидкість менша за нуль, то перед прокиданням виключення, додати в лог запис виду:2022-05-31 17:40:27,267 ERROR Horse: Speed is negative
  4. Якщо передана в конструктор дистанція менша за нуль, то перед прокиданням виключення, додати в лог запис виду:2022-05-31 17:41:21,938 ERROR Horse: Distance is negative
  5. Наприкінці конструктора додати у лог запис виду:2022-05-31 17:15:25,842 DEBUG Horse: створення Horse, ім'я [Лобстер], швидкість [2.8]

Логи повинні писатися у файл hippodrome.log , який має розташовуватися в корені проекту в папці logs . Кожен день файл повинен перейменовуватись за шаблоном у hippodrome.2021-12-31.log і замість нього повинен створюватися новий hippodrome.log . Для цього використовуй апендер RollingFile . При цьому файли старше 7 днів повинні видалятися. Для цього можеш використовувати конструкцію виду:

<DefaultRolloverStrategy>
    <Delete …>
        <IfFileName …/>
        <IfLastModified …/>
    </Delete>
</DefaultRolloverStrategy>

Погугли, що треба підставити замість три крапки. 😊
