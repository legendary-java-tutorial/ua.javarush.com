До масового поширення JavaScript був популярний формат зберігання та пересилання даних на основі XML.

Інформація про людину в цьому форматі могла виглядати так:

<person firstName="Bill" lastName="Gates">
   <birthday day="12" month="10" year="1965">
   <address city="Radmond" state="Washington" street="Gates 1" zipCode="93122">
   <phone home="+123456789" work="+123456799">
</person>

Такий код майже весь складався з тегів і дуже зручний для парсингу програмами. Проте людям читати такий код було складно. Тому згодом його почав витісняти формат JSON, створений на основі об'єктів JavaScript.

JSON і розшифровується JavaScript Object Notation.

Той самий об'єкт, записаний у вигляді JSON, буде виглядати так:

{
  "firstName": "Bill",
  "lastName": "Gates",
  "birthday": {
   	"day": "12",
   	"month": "10",
   	"year": "1965" },
  "address": {
   	"city": "Radmond",
   	"state": "Washington",
   	"street": "Gates 1",
   	"zipCode": "93122"},
  "phone": {
    "home": "+123456789",
    "work": "+123456799"}
}

Такий запис складніший для комп'ютера, але простіше для людини. Зі зростанням популярності інтернету та JavaScipt-а, зокрема, такий формат витіснив усі інші. Крім того, були написані швидкі бібліотеки-парсери JSON-даних.

У Java є бібліотеки, які можуть конвертувати Java-об'єкти в JSON-формат і назад. Так що тобі як Java-програмісту нема про що турбуватися.

Крім того, з JDK 7 Java з'явився вбудований тип даних - JsonObject. Докладніше по ньому ти можеш почитати у документації .
