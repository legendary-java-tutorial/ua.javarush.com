5.1 Додаємо Tomcat до IDEA Ultimate

Крок перший. Створюємо локальну конфігурацію Tomcat. Run - Edit Configurations -

Крок другий. Потім вибираємо локальний Tomcat.

Крок третій. Конфігуруємо Tomcat. Натиснувши на strong>configure, додаємо шлях до папки Tomcat.

Тут ти можеш вказати:

  • HTTP port- порт, на якому працюватиме Tomcat
  • JRE- можна вибрати JRE, на якій працюватиме Tomcat
  • VM options- Параметри віртуальної машини для Tomcat
  • URL— це посилання відкриватиме IDEA після запуску сервера за її допомогою

Крок четвертий. Додаємо наш проект як артефакт Tomcat.

Для цього переходимо на вкладку Deployment та натискаємо праворуч кнопку +.

От і все!

5.2 Створюємо перший веб-додаток

Якщо у тебе ще немає жодного веб-програми, то можна створити його в IDEA двома способами. Проект на основі Maven та просто рідний JavaEE-проект.

Якщо ти хочеш створити рідний web-проект від IDEA, то слідуй цій простій інструкції:

Крок 1 Створюємо новий проект ( menu File -> New Project), потім вибираємо:

  • Тип проекту - Java Enterprise
  • Шаблон (template) програми - Web Application
  • Application Server — поточний конфігурований Tomcat Server . Якщо він ще не доданий, то праворуч є кнопка New.
  • JDK - твою поточну Java JDK

Крок 4 . Твій проект готовий, можеш запускати його кнопкою Run чи Debug.

5.3 Створюємо перший веб-додаток на основі Maven

Якщо ти хочеш створити web-додаток на основі проекту Maven, то інструкція буде ще простішою.

Крок 1 Створюємо новий проект ( menu File -> New Project), потім вибираємо:

  • Тип проекту - Maven Archetype
  • JDK - задаємо JDK проекту
  • Archetype (шаблон проекту) - задаємо maven-archetype-webapp

Крок 2 Отримаємо проект такого типу:

Проект згенеровано, але Tomcat ще не налаштований. Щоб мати можливість запускати або дебажувати проект, тобі потрібно налаштувати Tomcat і додати свій проект до нього як артефакт. Як саме це робити ми вже розглядали раніше.
