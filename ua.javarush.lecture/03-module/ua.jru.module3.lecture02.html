Завдання: Потрібно зробити UI для CRUD бекенду, використовуючи HTML, CSS, JS, jQuery. Робитимемо адмін-панель для управління акаунтами онлайн-ігри.

  1. Зробити fork із репозиторію: https://github.com/vasylmalik/project-front.git
  2. Завантажити свою версію проекту на комп'ютер.
  3. Завантажити собі Tomcat: https://tomcat.apache.org/download-90.cgi .Важливо: Tomcat потрібен саме версії 9 , а не 10
  4. Налаштувати запуск програми через Ідею: Alt + Shift + F9 -> Edit Configurations… -> Alt + insert -> tom (у рядок пошуку) -> Local.
    • Після цього потрібно натиснути CONFIGUREі вказати, куди завантажено і розпаковано архів з Tomcat.
    • У вкладці Deployment: Alt + insert -> Artifact... -> rpg:war exploded -> OK.
    • У полі Application context: залишити тільки /(сліш).
    • Натиснути APPLY.
    • Закрийте вікно налаштування.
  5. Запустити програму. Для цього: Alt+Shift+F9 -> (ім'я конфігурації, я назвав 'App' у себе) -> Run.
  6. Після деплою програми відкриється нова вкладка у вибраному браузері при конфігурації. Відмінно, сервер налаштований, далі потрібно працювати з двома файлуми:
    • <project_dir>/src/main/webapp/html/my.html– тут буде розмітка та скрипти.
    • <project_dir>/src/main/webapp/css/my.css- Тут буде стилі. Все щодо стилів – на твій розсуд, жодних вимог до них немає.
  7. У розмітку ( my.html ) додати таблицю (не забудь їй вказати id), в якій вказати назви колонок:
    • #
    • Name
    • Title
    • Race
    • Profession
    • Level
    • Birthday
    • Banned
  8. Додати парний тег <script>– тут писатимемо функції.
  9. Перша функція повинна надсилати GET запит на URL "/rest/players"– отримати список. Результат потрібно додати до таблиці з п.7 (ось і став у нагоді id таблиці). Якщо все зробив правильно – після перезапуску сервера у тебе на сторінці відображатиметься таблиця на 4 рядки. У першому рядку – імена колонок, у решті – дані, які прийшли з сервера.
  10. Тепер, бажано, додати таблиці межі осередків, бо інформація взагалі нечитана.
  11. Тепер візьмемося пейджингом. Для цього потрібно:
    • Під таблицею додати секцію, наприклад, <div>у якій будуть кнопки пейджингу.
    • Додати функцію, яка надішле GET запит на сервер, який повертає загальну кількість облікових записів на сервері. URL: "/rest/players/count".
    • Додати список, що випадає, на 3-4 числових значення, скільки показувати акаунтів за раз. Числа мають бути в діапазоні від 3 до 20 включно.
    • У способі відображення списку облікових записів (п.9) додамо розрахунок кількості сторінок, які потрібні при заданій кількості облікових записів на сторінку, щоб можна було переглянути всі облікові записи.
  12. У функцію показу списку облікових записів додай параметр, який буде відповідати за номер сторінки, яку показуватиме. Відкоригуй URL, щоб він параметрами запиту передавав "pageNumber" та "pageSize" . Тепер при зміні кількості на сторінку в списку, що випадає, повинна показуватися запитана кількість акаунтів:
  13. Тепер давай підфарбуємо номер поточної сторінки, а то незручно.
  14. Додамо до таблиці ще 2 колонки: Edit та Delete. Для кожного рядка давай додамо іконки, які відповідатимуть за редагування та видалення кожного запису. Можеш використовувати або картинки, які є в <project_dir>/src/main/webapp/img/, або використовуй свої.
  15. Напиши функцію, яка відповідатиме за видалення облікового запису. Для цього потрібно надсилати DELETE запит на сервер на URL "/rest/players/{id}". При натисканні на картинку кошика – викликай цю функцію. Після дзвінка – не забудь оновлювати список облікових записів на поточній сторінці.
    • До видалення облікового запису з id=23
    • Після видалення облікового запису з id=23 . Зверніть увагу, жодних кнопок, крім кошика, не натискалося.
  16. Тепер напиши функцію, яка відповідає за редагування облікового запису. При натисканні на неї давай приховаємо кнопку "Delete", а картинку кнопки "Edit" поміняємо на "Save".
  17. Тепер додамо функціонал для редагування облікового запису. Натисніть кнопку “Edit”, крім зміни картинки, потрібно робити поля редагованими:
    • Name
    • Title
    • Race
    • Profession
    • Banned

    Зверніть увагу, що значення в полях для редагування мають бути ті, що є в обліковому записі.

  18. Тепер зробимо відправлення змін на кліку на кнопку “Save”. Для цього потрібно надіслати POST запит на URL “/rest/players/{id}”і як тіло запиту передати значення всіх п'яти полів, які можна змінювати у вигляді JSON. Використовуй JSON.stringify({…}); Після зміни, не забудь перезавантажити дані на поточній сторінці.
  19. Отже, із CRUD на даний момент має бути зроблено все, окрім “C” (Create). Після кнопок пейджингу, додай горизонтальну лінію та текст про те, що тут можна створити новий обліковий запис. Крім цього, додай блоки тексту та поля для введення для параметрів облікового запису:
    • Name – текст, від 1 до 12 символів.
    • Title – текст від 1 до 30 символів.
    • Race – список, що випадає, допустимі значення дивися в API.
    • Profession – список, що випадає, допустимі значення дивися в API.
    • Level – число від 0 до 100.
    • Birthday - дата (<input type = "date" ...).
    • Banned – булеве значення.
  20. Тепер додай кнопку, на кліку на яку всі введені дані будуть відправлятися на сервер. Використовуй POST запит на URL /rest/players. Після відповіді сервера, не забудь очистити поля для введення даних і перезапитати список облікових записів на поточній сторінці.
  21. Тепер можна додати "краси" через стилі - тут все на твій розсуд. У мене вийшло так:

Create player

URL /rest/players
Method POST
URL Params None
Data Params 
{
  “name”:[String],
  “title”:[String],
  “race”:[Race],
  “profession”:[Profession],
  “birthday”:[Long],
  “banned”:[Boolean], --optional, default=false
  “level”:[Integer]
}
Success Response 
Code: 200 OK
Content: {
  “id”:[Long],
  “name”:[String],
  “title”:[String],
  “race”:[Race],
  “profession”:[Profession],
  “birthday”:[Long],
  “banned”:[Boolean],
  “level”:[Integer]
 },
Notes

Ми не можемо створити гравця, якщо:

  • вказані не всі параметри Data Params (крім banned);
  • довжина значення параметра “name” або “title” перевищує розмір відповідного поля (12 та 30 символів);
  • значення параметра “name” пустий рядок;
  • рівень знаходиться поза заданими межами (від 0 до 100);
  • "birthday": [Long] < 0;
  • дата реєстрації знаходяться поза заданими межами.

У разі всього перерахованого вище - відповідь помилкою з кодом 400.
