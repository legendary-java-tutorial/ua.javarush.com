----------------------------------------

Примітка перекладача: цю статтю я виявив на ресурсі javareviseted і вважав добірку автора досить повною та цікавою. У ній здебільшого розказано про нові книги (або нові видання). Причому йдеться не тільки про Java, але також про суміжні технології та про Soft Skills. Хочу зазначити, що інформація у цій добірці зацікавить, скоріше, не абсолютних новачків, а тих, хто пройшов приблизно половину курсу CodeGym. Або більше. Для тих, хто тільки починає і шукає підручник з основ програмування на Java, рекомендую придивитися до моєї статті «Топ-7 книг з Java для початківців» . У цьому перекладі намагався наводити посилання російськомовне видання, якщо воно є у природі. Усім успішного навчання! А ось і список книг, які наводить автор оригінального матеріалу:

1. Java 8 in Action

Підручник з програмування на Java 8. Це одна з книг, які я читаю, ще починаючи з 2016 року. в руки.

2. Clean Architecture

( Мартін Р. "Чиста архітектура. Мистецтво розробки програмного забезпечення" ) Відмінна книга, виявлена ​​мною в 2017 році. Я справжній фанат книг дядечка Боба і вже прочитав Clean Code ("Чистий код") та The Clean Coder ("Ідеальний програміст") , присвячені правильній побудові архітектури. Якщо ви хочете розібратися в перевагах та недоліках різних архітектур програмного забезпечення – ця книга для вас.

3. Grokking Algorithms

( Бхаргава А. "Грокаємо алгоритми" ) Ще одна прекрасна книга, прочитана мною в 2017, і, ймовірно, найкраща книга 2017 року. Хоча вона охоплює лише невелику кількість алгоритмів та структур даних, все охоплене – винятково цікаво та корисно. Цей підручник із програмування алгоритмів співвідносить старі поняття із сучасними прикладами, вдихаючи в них нове життя, наприклад, описуючи, як Facebook зберігає дані про користувачів. Якщо коротко, це книга, яку просто зобов'язаний прочитати будь-який вивчати алгоритми в 2018 році.

4. Building Microservices: Designing Fine-Grained Systems

( Ньюмен С. "Створення мікросервісів" ) Все навколо тільки і говорять про мікросервіси, але що це таке? Яка головна перевага архітектури мікросервісів над монолітною? Ця книга відповідає на ці запитання. Я ще не закінчив її читання, тому збираюся читати її і в 2018.

5. Soft Skills

( Сонмез Д. Шлях програміста ) Це не самовчитель із програмування, це самовчитель «Як бути програмістом, щоб тебе не ненавиділи інші люди». Багато розробників ігнорують так звані "навички міжособистісного спілкування" (soft skills), наприклад, написання повідомлень електронної пошти, акцент на кар'єрному зростанні та самовдосконаленні. Джон Сонмез написав чудову книгу, і якщо ви відчуваєте, що ваша кар'єра зупинилася, і хотіли б її підштовхнути у 2018 році, прочитайте її обов'язково.

6. Database Design for Mere Mortals

(Проектування баз даних для простих смертних) Старий добрий підручник із програмування, а точніше – проектування баз даних. Я люблю знаходити хороші старі книги, які не втратабо актуальності, і це одна з них. Це все ще відмінна відправна точка для тих, хто хоче вивчити проектування баз даних або перевести на новий рівень свої навички їхнього моделювання.

7. Making Java Groovy, by Ken Kousen

(Groovy для Java) У 2017 році мені довелося трохи попрацювати з мовою програмування Groovy і це була перша прочитана мною книга. Якщо ви вирішите в 2018 році вивчити Groovy, вона забезпечує відмінну базу знань з точки зору Java-програміста. У ній наведено стільки подробиць, скільки потрібно для використання всіх його переваг – і нічого зайвого і незрозумілого для новачків.

8. Groovy in Action, Second Edition

(Groovy у дії) Це була друга прочитана мною у 2017 про Groovy книга. Хоча "Making Java Groovy" і непогана для початку, вона охоплює далеко не всі питання, тому після вивчення основ Groovy і написання декількох сценаріїв на ньому, вам знадобиться докладніша інформація. Саме тут вам і стане в нагоді ця книга.

9. TCP/IP Illustrated

(TCP/IP в картинках) Одна з книг, що найменш сподобалися мені, але все ж дуже корисна. Я ще не дочитав її, але якщо вам потрібні всі подробиці про TCP/IP – її можна буде почитати.

10. UML Distilled by Martin Fowler

( Мартін Фаулер, "UML. Основи. 3-те видання" ) У 2017 мені довелося намалювати кілька UML-схем, і щоб освіжити свої знання в UML я вибрав саме цю книгу. Стане в нагоді вона вам і для вивчення UML у 2018-му.

11. Hibernate Tips

(Поради з Hibernate) Одна з найкорисніших книг з Java 2017 року, написана відомим блогером Торбеном Янссеном (Thorben Janssen). У книзі наведено 70 практичних порад з Hibernate. Якщо ви використовуєте Hibernate , ці поради допоможуть заповнити прогалини у ваших знаннях, та й якщо ви вирішите вивчати Hibernate у 2018 році, з цією книгою ваше навчання набуде нового виміру.

12. The Art of Agile Development

(Мистецтво розробки на Agile) Я використовую Agile вже досить давно, але, як і раніше, читаю іноді книги по ньому, щоб освіжити знання і вивчити щось нове. Ця книга виявилася відмінним керівництвом з Agile, навіть ті, хто використовує його, регулярно зможуть почерпнути звідси що-небудь.

13. Essential Scrum

(Основи Scrum) Я грав роль скрам-майстра (Scrum Master) у 2017 році та для підготовки читав саме цю книгу. Scrum чудово працює для невеликих, локальних команд розробників, але якщо намагатися керувати великою командою, розкиданою по всьому світу, віддалено, то щось швидко може піти не так. У цій книзі описані всі утиліти та дано всі інструкції, необхідні для організації скрам-нарад (Scrum meetings) – все, що потрібно, щоб стати скрам-майстром. Якщо ви хотіли б у 2018 році стати скрам-майстром – прочитайте її. Можете також прослухати будь-які онлайн-курси по Scrum і Agile, для чого рекомендую вам заглянути в Scrum Fundamentals на сайті Pluralsight.

14. Java Performance Companion

(Довідник з продуктивності Java). Підручник з Java для тих, хто вже щось знає. Я прочитав кілька книг з тонкого настроювання продуктивності Java в 2016 році і раніше, наприклад, The Definitive Guide to Java Performance (Вичерпний посібник з продуктивності Java), так що багато чого від неї я не очікував. Єдиною моєю метою було розібратися зі збирачем сміття (garbage collector) G1, про яке я звідти дізнався. Ця книга – відмінний відправний пункт для тих, хто хотів би в 2018 зосередити свої зусилля на тонкому налаштуванні продуктивності Java.

15. High-Performance Java Persistence

(Високопродуктивне зберігання даних у Java) Ще одна книга, що залишилася з 2016 року, яку я дочитав у 2017. І ще одна прекрасна книга з Hibernate з упором на продуктивність. Якщо ви зібралися всерйоз дізнатися більше про Hibernate у 2018, рекомендую вам прочитати як книгу з п.11, так і цю.

16. Functional Programming in Scala

(Функціональне програмування на Scala) Я намагався розібратися з мовою Scala у 2017 році, але без особливих успіхів. Я прочитав пару книг по ньому, але я не мав недостатньої практики для справжнього вивчення. Зокрема, тому що я не використовую його у своїх проектах і маю важливіші речі, яким варто приділити увагу. У будь-якому випадку, якщо ви вирішите зайнятися функціональним програмуванням та мовою Scala у 2018 – завітайте до цієї книги. Можливо, я перечитаю її у 2018 році, якщо буде час.

17. Scala for the Impatient

(Scala для нетерплячих) Ще одна прочитана мною у 2017 книга з мови Scala. Я обожнюю стиль Кея С. Хортсманна і прочитав кілька його книг, зокрема Core Java Part 1 and II (Java. Бібліотека професіонала) та Java SE 8 for the Really Impatient (Java SE 8 базовий курс). Це книга в подібному стилі, присвячена мові Scala. Це непоганий початковий посібник для тих, хто збирається вчити Scala у 2018.

18. Head First JavaScript

( Робсон Е., Фрімен Е. "Вивчаємо програмування на JavaScript" ) Я вже говорив, що JavaScript - мова програмування №1 на сьогодні, так що якщо ви його не знаєте - ви багато втратабо. Ця книга є обов'язковою для читання для всіх веб-розробників і вивчають такі фреймворки, як Angular, React і jQuery. Якщо ви ще тільки маєте вивчити JavaScript - немає краще книги, ніж Head First JavaScript. Обов'язкова до прочитання для всіх, хто вирішив у 2018 вивчити JavaScript.

19. SQL CookBook

( Молінаро Е. "SQL. Збірник рецептів" ) SQL - моя улюблена технологія, я завжди із задоволенням знаходжу цікаві книги з SQL. Цю книгу я знайшов у 2017 і радий поділитись нею з вами. Ця книга орієнтована на практику, в ній наводиться безліч відмінних порад щодо використання SQL у різних базах даних. Прочитайте її в 2018 році, якщо хочете підняти свої навички SQL на новий рівень. Ви також можете скористатися цими веб-сайтами з вивчення SQL-запитів для подальшого вдосконалення своїх знань у SQL.

20. The Complete Software Developer's Career Guide

(Всеосяжний посібник з кар'єрного зростання розробника програмного забезпечення) Остання за списком, але не по важливості, відмінна книга для справжніх розробників ПЗ. Якщо ви всерйоз налаштовані на кар'єрне зростання і хотіли б взяти справу у свої руки в 2018, то обов'язково прочитайте цю книгу. Джон Сонмез (John Sonmez) із сайту SimpleProgrammer.com дає в цій книзі і згадуваному вище "Шляхи програміста" безліч корисних порад.
