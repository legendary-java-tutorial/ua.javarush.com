Іноді з'являється необхідність запускати завдання у певний час. Часто, для вирішення цієї проблеми, використовую Cron Jobs у Unix системах. Але цей підхід незручний у тому плані, що при міграції з сервера на сервер легко забути про Cron тасках. Тяжко їх профілювати, тестувати, зберігати, стежити за змінами, відстежувати статуси завдання тощо. У проектах Java та Spring Boot, можна використовувати Schedluled Jobs. У відео уроки, в режимі live coding, створимо Spring Boot додаток, який буде ширити новини за розкладом, зберігати дані в базі даних, виводити новини використовуючи REST API, і розділимо це все на шари для масштабованої архітектури проекту.
