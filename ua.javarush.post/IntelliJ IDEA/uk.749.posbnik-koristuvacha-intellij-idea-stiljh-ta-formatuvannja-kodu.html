Стиль та форматування коду

Стандарт оформлення коду визначає стиль відступів, використання прогалин при оформленні виразів, спосіб розміщення дужок, що обмежують логічні блоки, порожні рядки, порядок операторів і т.д.

  1. Форматування коду

    IntelliJ IDEA автоматично застосовує стиль коду, налаштований під час редагування, і в більшості випадків вам не потрібно викликати Reformat Code (Форматування коду) явно. Однак, ви можете зробити це в будь-який час як для всього файлу, так і просто для вибраного шматка коду, або навіть для всього каталогу, просто натиснувши Alt+Ctrl+L (Alt+Cmd+L для Mac.)

    Якщо ви бажаєте застосувати Reformat Code для всіх файлів у директорії, використовуйте ту ж кнопку на пенелі інструментів проекту.

  2. Автоматичне вирівнювання рядків

    Ще одна корисна можливість Auto-Indent Lines , яка допомагає вам виправити відступ рядка. Ця дія іноді є кращим варіантом, ніж Reformat Code , тому що він не вимагає від вас вибрати щось. Просто натисніть Alt + Ctrl + I ( Alt + Cmd + I для Mac) , і відступи для поточного рядка буде швидко впорядковано.

  3. Налаштування

    IntelliJ IDEA дозволяє вам налаштувати параметри стилів коду для кожної з мов, що підтримуються, або для проекту або для середовища розробки в цілому за допомогою Settings → Code Style

    Схеми стилів для проекту можуть бути поширені серед інших членів вашої команди розробників за допомогою системи контролю версій.

  4. Символ табуляції

    Остання, але не менш важлива настройка, що стоїть вашої уваги, це Use tab character (використання символу табуляції). За замовчуванням ця настройка вимкнена, і IntelliJ IDEA використовує звичайні пробіли для вирівнювання рядків замість табуляції символів. Якщо у файлух використовується занадто багато пробілів для вирівнювання коду, ви можете оптимізувати їх розмір увімкнувши це налаштування (4 пробіли будуть замінені на 1 символ табуляції).
