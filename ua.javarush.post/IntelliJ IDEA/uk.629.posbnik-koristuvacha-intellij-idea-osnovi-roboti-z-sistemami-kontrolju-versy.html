Wikipedia каже, що " Система управління версіями (від англ. Version Control System, VCS або Revision Control System) - програмне забезпечення для полегшення роботи з інформацією, що змінюється. Система управління версіями дозволяє зберігати кілька версій одного і того ж документа, при необхідності повертатися до більш раннім версіям, визначати, хто і коли зробив ту чи іншу зміну, та багато іншого."

  1. Щоб отримати ваш проект із системи контролю версій, потрібно у вітальному вікні IDEA вибрати Check out from Version Control або в меню, вгорі, VCS і там вивратити Git, Git Hub.

  2. Установки системи контролю версій доступні меню File - other settings - Default settings - Version control. Ви можете вибрати кореневу директорію проекту для вашої VCS. Ще ви завжди можете вимкнути VCS.

  3. Після підключення VCS на проекті ви можете бачити та керувати змінами. Cmd + 9 для Mac для одержання вікна Changes.

  4. Якщо вам потрібно зробити якусь операцію з VCS, можна викликати спливаюче меню операцій Ctrl + V для Mac. Наприклад, якщо ви хочете закомітити зміни.

  5. Ви можете відстежувати історію змінення файлів. Для цього потрібно в меню VCS – Git (or your any VCS) – Show history або у спливаючому меню, яке з'являється після натискання правою кнопкою вибрати Git (or your VCS) – Show history або Ctrl+V для Mac – Show history.

    У вкладці Log, з права внизу, ви можете подвійним клацанням відкрити змінений файл і побачити конкретні зміни в ньому. Зелененьким зазначено, що додалося.

  6. Анотації дозволяють бачити хто і коли змінив кожен конкретний рядок коду. Анотації доступні з трьох варіантів меню: спливаюче (Ctrl + V для Mac), контекстне (правої кнопкою) і основне меню VCS на панелі.

  7. Ви можете налаштувати список файлів, що ігноруються. Наприклад, використовуючи кнопку Show ignored files в Changes

  8. У Git за допомогою IDEA ви можете створювати, перемикатися, об'єднувати, порівнювати та видаляти гілки коду. Щоб побачити список існуючих гілок або створити нову, використовуйте або Branches з головного або контекстного меню, спливаюче меню VCS operations, або елемент керування на правій стороні панелі статусу. (Пункт взято з посилання )

  9. Patche дозволяє зберегти набір змін у вигляді файлу, який можна передати електронною поштою і потім застосувати до коду. Це корисно, якщо ви працюєте віддалено, без постійного підключення до системи контролю версій, але все одно повинні передавати ваші зміни іншим членам проекту. (інформація з посилання )

    Це можна зробити правою кнопкою миші по зміненому файлу в розділі Changes і вибрати Create Patch.

    Потім ви можете поставити галочки навпроти конкретних файлів, які хочете зберегти файл.
